# Hermes Agent 学习版架构梳理文档
> **星级定义**：
> ★★★★★ 核心必学：Agent核心逻辑，必须彻底掌握
> ★★★★ 重要必学：核心配套组件，理解Agent工作原理必须掌握
> ★★★ 可选学习：扩展功能，学完核心之后按需学习
> ★★ 不重要：工具类、辅助脚本，不需要深入理解，会用就行
> ★ 可剔除：和核心Agent逻辑无关，学习时可以完全忽略/删除

---
## 🎯 整体架构分层（从核心到边缘）
```
┌──────────────────────────────────────────────────────────┐
│  接入层（终端/API/微信/邮件等）  ★★★ 可选               │
├──────────────────────────────────────────────────────────┤
│  技能层（复杂流程化能力）       ★★★★ 重要必学           │
├──────────────────────────────────────────────────────────┤
│  工具层（原子能力调用）         ★★★★ 重要必学           │
├──────────────────────────────────────────────────────────┤
│  核心Agent层（推理/记忆/循环）  ★★★★★ 核心必学          │
├──────────────────────────────────────────────────────────┤
│  基础依赖层（模型调用/配置/工具）★★★★ 重要必学           │
└──────────────────────────────────────────────────────────┘
```

## 📦 模块详细梳理
### ✅ ★★★★★ 核心必学模块（必须100%掌握，不建议删除）
| 模块路径 | 作用 | 学习重点 |
|----------|------|----------|
| `agent/` 目录 | 整个项目的大脑，Agent核心逻辑实现 | 1. `agent/agent.py` Agent主类，核心执行循环<br>2. `agent/memory.py` 记忆系统实现，长短记忆管理<br>3. `agent/tool_call.py` 工具调用解析、执行、结果处理<br>4. `agent/prompt_builder.py` 动态Prompt组装逻辑<br>5. `agent/skill_executor.py` 技能执行调度 |
| `run_agent.py` | 最简化的Agent启动入口，100行代码就能跑通整个Agent流程 | 这是初学者入门的最佳入口，直接打断点调试看完整执行流程：用户输入→Prompt组装→大模型调用→工具调用判断→工具执行→生成回复 |
| `model_tools.py` | 大模型调用抽象层，统一适配OpenAI/Anthropic/Mistral等不同模型API | 学习怎么封装不同厂商的大模型接口，解耦Agent逻辑和具体模型实现 |

---
### ✅ ★★★★ 重要必学模块（理解Agent生态必须掌握，建议保留）
| 模块路径 | 作用 | 学习重点 |
|----------|------|----------|
| `tools/` 目录 | Agent原子能力库，所有Agent能调用的外部能力都在这里封装 | 1. `tools/base.py` 工具基类定义，理解工具的标准接口<br>2. 看简单工具示例：`tools/calculator_tool.py`、`tools/search_tool.py`<br>3. 自己尝试写一个新工具（比如获取当前天气的工具） |
| `skills/` 目录 | Agent高级流程化能力，比工具更复杂的多步任务实现 | 1. `skills/base.py` 技能基类定义<br>2. 理解技能和工具的区别：工具是单次调用，技能是多步、有状态的复杂任务<br>3. 尝试写一个简单技能（比如"自动总结网页内容"技能，调用搜索工具+爬取工具+总结能力） |
| `hermes_state.py` / `hermes_constants.py` | 全局状态、配置管理 | 理解系统怎么管理配置、会话状态、持久化数据 |
| `toolsets.py` | 工具集管理，按需加载不同的工具组合给Agent | 学习怎么给不同场景的Agent分配不同的工具权限 |

## 🎯 最小可运行核心子集（最轻量化的学习版本）
如果你想极致精简，只保留最核心的Agent逻辑，只需要以下几个文件，总共不到1000行代码，就能跑通一个完整的、支持工具调用、记忆的Agent：
```
├── agent/                # 核心Agent逻辑
│   ├── __init__.py
│   ├── agent.py
│   ├── memory.py
│   ├── tool_call.py
│   └── prompt_builder.py
├── tools/                # 简单工具示例
│   ├── __init__.py
│   ├── base.py
│   └── calculator_tool.py
├── model_tools.py        # 大模型调用封装
├── run_agent.py          # 启动入口
├── pyproject.toml        # 仅保留核心依赖：openai/anthropic SDK、pydantic、python-dotenv
└── .env                  # 配置大模型API Key
```

## 📚 学习路线建议（从易到难）
1. **第一步（1天）**：从 `run_agent.py` 开始跑通项目，打断点走一遍完整的对话流程，理解Agent的核心执行循环。
2. **第二步（2-3天）**：深入看 `agent/` 目录下的核心代码，搞懂记忆系统、工具调用、Prompt组装的实现原理。
3. **第三步（1-2天）**：学习工具系统，自己写一个简单的工具（比如获取当前时间、查询天气的工具），让Agent能成功调用它。
4. **第四步（2-3天）**：学习技能系统，写一个简单的多步技能（比如"自动搜索某个主题，总结前3个网页内容"的技能）。
5. **第五步（按需）**：学习扩展内容，比如网关接入、插件系统、定时任务等。

> 💡 学习小技巧：把所有不相关的代码都删掉，只保留核心子集，代码量小了之后更容易理解，不会被无关内容干扰。
